Basic Prometheus Queries
System Health:

# CPU usage
100 - (avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# Memory usage
(1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100

# Disk usage
100 - ((node_filesystem_avail_bytes * 100) / node_filesystem_size_bytes)
Application Metrics:

# HTTP request rate
rate(http_requests_total[5m])

# Response time percentiles
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))

# Error rate
rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m])
ML Model Performance:

# Prediction latency
ml_prediction_duration_seconds

# Model accuracy
ml_model_accuracy_ratio

# Predictions per second
rate(ml_predictions_total[1m])
Security Metrics:

# Threat detection rate
rate(threats_detected_total[5m])

# Risk score distribution
histogram_quantile(0.90, rate(risk_score_bucket[5m]))

# Blocked IPs count
security_blocked_ips_total
Prometheus CLI Commands
Query data:

# Query current CPU usage
curl 'http://localhost:9090/api/v1/query?query=up'

# Query with time range
curl 'http://localhost:9090/api/v1/query_range?query=rate(http_requests_total[5m])&start=2024-01-01T00:00:00Z&end=2024-01-01T01:00:00Z&step=15s'
Configuration:

# Check Prometheus config
curl http://localhost:9090/api/v1/status/config

# Reload config
curl -X POST http://localhost:9090/-/reload

# Check targets
curl http://localhost:9090/api/v1/targets
Alerting:

# High CPU alert
up == 0

# High error rate
rate(http_requests_total{status=~"5.."}[5m]) > 0.1

# ML model down
absent(ml_predictions_total)
These should work with your Docker monitoring stack setup.
